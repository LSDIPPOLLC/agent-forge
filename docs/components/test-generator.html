Agent Forge

TestGenerator

Auto-generates diverse test cases from an agent spec, with intent tracing and diversity enforcement.

Interface

import { TestGenerator } from "agent-forge/test_generator";
import { createOllamaBackend } from "agent-forge/llm_backend/ollama";

const llm = createOllamaBackend();
const tg = new TestGenerator(llm);

const testCases = await tg.generate({
  description: "review PRs for logic errors and style violations",
  inputSchema: { pr_url: "string", repo: "string" },
  outputSchema: { issues: "array", score: "number", summary: "string" },
  tools: ["gh_api", "read_file"],
  k: 5,
});

const coverage = tg.checkCoverage(testCases, description);
// coverage.gaps = ["style violations"] if not covered

TestCase Structure

interface TestCase {
  id: string;
  input: unknown;
  expectedSchema: { keys: string[] };
  tracedTo: string[];       // description clauses this test covers
  diversityScore: number;   // 0.0-1.0, how unique this test is
}

Diversity Enforcement

After parsing the LLM-generated test cases, pairwise Jaccard similarity is computed:

function jaccardSimilarity(a: unknown, b: unknown): number {
  const aStr = JSON.stringify(a);
  const bStr = JSON.stringify(b);
  if (aStr equals bStr) return 1;
  const aSet = new Set(aStr.split(""));
  const bSet = new Set(bStr.split(""));
  const intersection = [...aSet].filter(c equals bSet.has(c)).length;
  const union = new Set([...aSet, ...bSet]).size;
  return intersection / union;  // greater than 0.7 = too similar, rejected
}

If fewer than k diverse tests are generated, fallback synthetic tests are appended.

Intent Tracing

The NL description is parsed for key aspects using regex patterns:

const aspectPatterns = [
  { pattern: /logic\s*errors?/gi, aspect: "logic errors" },
  { pattern: /style\s*violations?/gi, aspect: "style violations" },
  { pattern: /test\s*coverage/gi, aspect: "test coverage" },
  { pattern: /security/gi, aspect: "security" },
  { pattern: /performance/gi, aspect: "performance" },
];

Each test's tracedTo field maps it to the aspects it exercises. checkCoverage() then identifies any aspect with zero test coverage.

CoverageReport

interface CoverageReport {
  aspects: string[];         // all aspects found in description
  coveredAspects: string[];  // aspects with tests
  gaps: string[];             // aspects with 0 tests - BLOCKS export
}
