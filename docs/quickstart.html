Agent Forge

Quick Start

Create your first agent in under 5 minutes.

1. Create an Agent

agent-forge create "review PRs for logic errors and style violations"

The CLI will prompt for:

  • Output target — Claude Skills or Standalone
  • Tools — comma-separated list (default: read_file, run_command)
  • Pass threshold — 0.0 to 1.0 (default: 0.85)
  • Max iterations — refinement loop limit (default: 5)

2. Watch the Loop Run

[1/5] Generating agent...
[2/5] Evaluating...
[2/5] Score: 0.62 - refining...
[3/5] Refining agent (attempt 3)...
[3/5] Evaluating...
[3/5] Score: 0.71 - refining...
[4/5] Refining agent (attempt 4)...
[4/5] Evaluating...
[4/5] Score: 0.88 - agent passed quality gate

3. Inspect the Result

# List all agents
agent-forge list

# Inspect version history
agent-forge history pr_reviewer

# Inspect specific version
agent-forge inspect pr_reviewer --v 3

4. Use the Agent

If exported to Claude Skills, the agent is available at ~/.claude/skills/pr_reviewer/.

If exported as standalone:

cd agents/pr_reviewer
bun run agent.ts sample input here

Programmatic Usage

import runLoop from "agent-forge/runloop";

const result = await runLoop({
  description: "review PRs for logic errors",
  name: "pr_reviewer",
  tools: ["gh_api", "read_file"],
  threshold: 0.85,
  maxIterations: 5,
  outputMode: "claude-skills",
});

if (result.passed) {
  console.log("Agent created at: " + result.path);
} else {
  console.log("Best score: " + result.bestScore);
}

What Happens Inside the Loop

  1. Generate — LLM produces a TypeScript agent file from NL description
  2. Test Generation — LLM creates k=5 diverse test cases traced to intent clauses
  3. Evaluate — Agent runs against each test; LLM judge scores semantic quality
  4. Quality Gate — Composite score (60% tests + 40% judge) checked against threshold
  5. Refine — If gate fails, LLM refiner rewrites targeted parts of the agent
  6. Repeat — Loop until pass or max iterations reached
Tip: Start with a narrow, specific task. "Review PRs for logic errors" works better than "be a code expert."
