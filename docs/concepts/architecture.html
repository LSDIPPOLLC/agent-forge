Agent Forge

Architecture

Agent Forge is organized around a single orchestration loop driven by LLM-generated code, tests, and refinements.

System Overview

Natural Language Description
         |
   LoopController
    |          |
 Generator  ---- (on failure)
    |
 Agent File (single .ts)
    |
 Evaluator
    | TestGenerator (LLM auto-generates test cases)
    | TestRunner (exec agent against tests)
    | LLMJudge (semantic quality scoring)
    | ScoreReport
    |
 QualityGate: pass? - OutputAdapter - write to target
            fail? - Refiner - rewrite - loop

Components

ComponentRole
GeneratorNL description to AgentSchema to TypeScript agent file
TestGeneratorLLM generates k diverse test cases from agent spec
EvaluatorRuns agent against tests + LLM judge to ScoreReport
RefinerFailure report to targeted rewrite of agent
QualityGatePass/fail threshold with coverage and generalization checks
LoopControllerState machine orchestrating the full loop
RegistrySQLite persistence: agents, versions, eval history
RuntimeAgent execution via bun:eval subprocess
NestedExecutorMulti-agent call graph with depth limit + cycle detection
OutputAdapterWrites finished agents to Claude Skills or Standalone format

State Machine

IDLE - GENERATING - EVALUATING - REFINING - (loop or DONE)
                                          |
                                        FAILED
  • IDLE — Initial state, waiting for task
  • GENERATING — LLM producing initial agent code
  • EVALUATING — Running tests + LLM judge
  • REFINING — LLM rewriting agent from failure report
  • DONE — Agent passed QualityGate
  • FAILED — Max iterations reached without passing

LLM Backend Abstraction

All LLM calls go through LLMBackend interface. Currently supported:

  • Ollama (default) — Local, no API costs
  • OpenAI — GPT-4o, GPT-4o-mini, o3
  • Anthropic — Via OpenAI-compatible adapter

Set via environment variables — no code changes needed.
