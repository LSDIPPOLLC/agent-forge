Agent Forge

Overfitting Safeguards

Agent Forge includes multiple layers to prevent agents from optimizing to pass a narrow test suite instead of serving the intended use case (Goodhart's Law).

The Problem

When a measure becomes a target, it ceases to be a good measure.

— Goodhart's Law

If the TestGenerator creates 5 nearly identical test cases about null pointers, the agent can "pass" without learning to detect race conditions, style violations, or any other aspect of the real use case.

Safeguard Layers

1. Diversity Enforcement (TestGenerator)

After generating k test cases, pairwise Jaccard similarity is computed on inputs. Any test with similarity greater than 0.7 to an existing test is rejected and regenerated.

2. Intent Tracing (TestGenerator)

Every test case includes a tracedTo field mapping it to clauses in the original NL description. After test generation, checkCoverage() identifies any aspect of the description with zero test coverage. If "style violations" has no tests, a coverage gap is flagged.

3. Coverage Completeness (QualityGate)

If any aspect of the original intent has no test coverage:

  • A coverageGaps alert is added to the ScoreReport
  • QualityGate sets blocked to true
  • Export is halted until gaps are resolved

4. Generalization Holdout (Evaluator)

After an agent passes all generated tests, 3 shadow test cases are run against it. These are:

  • Within the intended scope of the agent
  • Not similar to any training test case
  • Used only to check whether the agent generalizes or memorizes

If generalization score less than 0.5, a warning is attached to the agent record. The agent still passes, but the warning persists.

5. Audit Trail (Registry)

Every agent version stores:

  • Which test cases it was trained on
  • Which test cases it was evaluated against
  • Full failure reports per iteration

This makes it possible to trace back: "Why did v3 pass but v4 fail?" or "Is this agent actually good at X or just good at these specific tests?"
