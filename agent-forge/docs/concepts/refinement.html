Agent Forge

Refinement Loop

When an agent fails the QualityGate, the Refiner rewrites it using the failure report as a targeted fix guide.

Loop Flow

describe("review PRs for logic errors")
    |
 Generator - agent_v1.ts
    |
 Evaluator - ScoreReport
    |  composite: 0.62, failures: [...]
    |
 QualityGate: FAIL (0.62 less than 0.85)
    |
 Refiner - agent_v2.ts (targeted fixes)
    |
 Evaluator - ScoreReport
    |  composite: 0.88
    |
 QualityGate: PASS - OutputAdapter - export

Refiner Strategy

The Refiner is not told to "rewrite everything." It receives:

  • Current agent code
  • Full failure report (test results, judge scores, specific failures)
  • System prompt instructing targeted, conservative changes

Common Fix Patterns

FailureFix
Missing output fieldsCheck OUTPUT_SCHEMA, ensure all keys returned
Schema mismatchAlign returned keys with expected schema exactly
Error handlingWrap operations in try/catch, return error info gracefully
Tool misuseVerify tool import and correct call signature
Edge case crashAdd null/undefined checks

Iteration Tracking

Every iteration is snapshot in the Registry:

  • Agent code at that version
  • Eval run results (score, failures, warnings)
  • Best score tracked across all versions

If the loop exits without passing, the best-scoring version is retained.

Max Iterations

Default: 5. Configurable via --max-iterations. After max:

  • Loop exits with FAILED state
  • Best version is saved in registry
  • Best code is available for inspection
  • Export is NOT triggered (requires passing gate)

Output on Export

When the agent passes, it's written to the target via an OutputAdapter:

  • Claude Skills: ~/.claude/skills/<name>/SKILL.md + agent.ts
  • Standalone: ./agents/<name>/agent.ts + README.md
