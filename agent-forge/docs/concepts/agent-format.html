Agent Forge

Agent Format

Agents are single TypeScript files with a structured header and a SKILL.md docstring block.

File Structure

// agent: pr_reviewer
// description: Reviews PRs for logic errors, style violations, and test coverage
// version: 3
// tools: gh_api, read_file, run_command
// input_schema: sample input schema
// output_schema: sample output schema

docstring block here (## SKILL.md)

import tools, handle from "agent-forge-runtime";

const INPUT_SCHEMA = { pr_url: "string", repo: "string" };
const OUTPUT_SCHEMA = { issues: "array", score: "number", summary: "string" };

async function handle(input: unknown): Promise<unknown> {
  const { pr_url, repo } = input as { pr_url: string; repo: string };
  // ... agent logic
  return { issues: [], score: 0, summary: "" };
}

export { handle };

Header Fields

FieldTypeRequiredDescription
agentstringYesUnique agent name (kebab-case)
descriptionstringYesOne-line description of purpose
versionnumberYesVersion number (incremented by Refiner)
toolsstring[]NoList of tools the agent uses
input_schemaRecordYesInput parameter names and types
output_schemaRecordYesOutput field names and types

SKILL.md Block

The docstring contains a ## SKILL.md section parsed by output adapters. It includes:

  • Role — one sentence persona
  • Tools — TypeScript signatures for each tool used
  • Behavior — detailed instructions for the agent

Handler Function

The handler function is the agent's entry point. It:

  • Receives input matching INPUT_SCHEMA
  • Uses tools from agent-forge-runtime
  • Returns output matching OUTPUT_SCHEMA
  • Handles errors gracefully, returning error info in output

Runtime Package

Agents import tools from agent-forge-runtime:

import tools, handle from "agent-forge-runtime";

Available tools:

  • read_file(path)
  • run_command(cmd, cwd?)
  • gh_api(endpoint, method?)
  • grep(pattern, path)
  • write_file(path, content)
  • search_web(query)
