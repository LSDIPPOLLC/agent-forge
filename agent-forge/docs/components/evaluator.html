Agent Forge

Evaluator

Runs an agent against test cases and scores quality using both schema validation and LLM judgment.

Interface

import { Evaluator } from "agent-forge/evaluator";
import { createOllamaBackend } from "agent-forge/llm_backend/ollama";

const llm = createOllamaBackend();
const evaluator = new Evaluator(llm);

const report = await evaluator.run({
  agentCode: agentCodeString,
  testCases: [
    {
      id: "tc-1",
      input: { pr_url: "https://github.com/org/repo/pull/1", repo: "org/repo" },
      expectedSchema: { keys: ["issues", "score", "summary"] },
      tracedTo: ["logic errors"],
      diversityScore: 1,
    },
  ],
  description: "review PRs for logic errors",
  outputSchema: { issues: "array", score: "number", summary: "string" },
});

// report.testPassRate: 0.0 - 1.0
// report.judgeScore: 0.0 - 1.0
// report.composite: 0.6 * testPassRate + 0.4 * judgeScore
// report.passed: boolean

ScoreReport Fields

interface ScoreReport {
  testPassRate: number;
  judgeScore: number;
  composite: number;
  generalizationScore?: number;
  generalizationWarning?: string;
  coverageGaps: string[];
  testResults: TestResult[];
  passed: boolean;
}

interface TestResult {
  testCase: TestCase;
  passed: boolean;
  output?: unknown;
  error?: string;
  executionTimeMs: number;
}

Execution

Each test case is executed via bun:eval in an isolated subprocess:

// Temporary files created per test
const agentFile = "`/tmp/agent-forge-evals/agent_` + runId + `.ts`";
const runtimeFile = "`/tmp/agent-forge-evals/runtime_` + runId + `.ts`";
const inputFile = "`/tmp/agent-forge-evals/input_` + runId + `.json`";

// Files cleaned up after execution
bun run agent_" + runId + ".ts

LLM Judge

The judge evaluates up to 3 test outputs (sampled) for semantic quality:

// Judge prompt instructs LLM to score 0-25 per criterion:
// - Correctness
// - Completeness
// - Quality
// - Format Compliance

// judgeScore = average(criterion_scores) / 25

Generalization Shadow Eval

After main tests, 3 shadow cases run to check generalization. These are synthetic inputs that are:

  • Within the intended scope
  • Not similar to training inputs

A warning is attached if the agent fails 2/3 or more shadow cases.

Customization

Key constants in src/evaluator.ts:

  • AGENT_RUNTIME_PACKAGE — the runtime shim injected for all agent executions
  • MAX_SHADOW_CASES — number of generalization holdout tests (default: 3)
  • EXEC_TIMEOUT_MS — per-test execution timeout (default: 30000ms)
