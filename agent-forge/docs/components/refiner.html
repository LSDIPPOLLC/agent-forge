Agent Forge

Refiner

Takes a failed evaluation and rewrites only the broken parts of the agent — targeted fixes, not full rewrites.

Interface

import { Refiner } from "agent-forge/refiner";
import { createOllamaBackend } from "agent-forge/llm_backend/ollama";

const llm = createOllamaBackend();
const refiner = new Refiner(llm);

const improvedCode = await refiner.refine({
  currentCode: agentCodeString,
  scoreReport: scoreReport,
  iteration: 2,
});

Refinement Prompt Strategy

The Refiner's system prompt instructs the LLM to:

  • Read the failure report carefully
  • Identify root causes, not just symptoms
  • Rewrite only the parts that need fixing
  • Keep what works — do not rewrite everything
  • Preserve the header comment block
  • Preserve INPUT_SCHEMA and OUTPUT_SCHEMA

Failure Report

The Refiner receives a human-readable failure summary:

Failed Tests (3/5):
  - Input: sample input - Missing required output field: "summary"
  - Input: sample input - Tool "gh_api" called incorrectly
  - Input: sample input - Error: Cannot read properties of undefined

Generalization Warning: Agent failed 2/3 generalization holdout tests
Coverage Gaps: style violations

Common Fix Patterns

PatternDetectionFix
Missing fieldsExpected keys not in outputAdd missing keys to return object
Schema mismatchOutput key names differAlign keys with OUTPUT_SCHEMA
Uncaught errorsSubprocess error in TestRunnerWrap logic in try/catch
Tool misuseTool returns unexpected shapeFix tool call arguments
Edge case crashNull/undefined accessAdd defensive checks

Code Extraction

After the LLM returns the refined code, the Refiner extracts the agent block by finding // agent: in the output. If extraction fails, the original code is returned with a console warning.

Temperature

The Refiner uses temperature 0.5 (lower than Generator's 0.7) to produce more deterministic, targeted changes rather than creative rewrites.
