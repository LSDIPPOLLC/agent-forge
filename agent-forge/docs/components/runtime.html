Agent Forge

Runtime

Handles agent execution in isolated subprocesses. Two implementations: Executor for single agents, NestedExecutor for multi-agent call graphs.

Executor

Executes a single agent against an input via bun:eval.

import { Executor } from "agent-forge/runtime/executor";

const executor = new Executor();

const result = await executor.execute(agentCode, {
  pr_url: "https://github.com/org/repo/pull/123",
  repo: "org/repo",
});

// result.output: unknown
// result.error: string or undefined
// result.executionTimeMs: number
// result.stdout: string
// result.stderr: string

Execution Flow

1. Create temp files:
   /tmp/agent-forge-runtime/agent_runId.ts
   /tmp/agent-forge-runtime/runtime_runId.ts
   /tmp/agent-forge-runtime/input_runId.json

2. Write agent code + runtime shim + input

3. Spawn: bun run agent_runId.ts

4. Capture stdout/stderr

5. Parse JSON output

6. Clean up temp files

Runtime Shim

Every agent execution injects a runtime package providing tool implementations:

export const tools = {
  read_file: async ({ path }) returns object,
  run_command: async ({ cmd, cwd }) returns object,
  gh_api: async ({ endpoint, method }) returns object,
  grep: async ({ pattern, path }) returns array,
  write_file: async ({ path, content }) returns void,
  search_web: async ({ query }) returns string,
};

NestedExecutor

Extends Executor with multi-agent support. Agents can call sub-agents via call_agent(name, input).

import { NestedExecutor } from "agent-forge/runtime/nested_executor";

const nested = new NestedExecutor();

const result = await nested.executeTopLevel("orchestrator_agent", {
  task: "analyze this PR",
});

// result.output: unknown
// result.callGraph: string[]  // agents in the call chain
// result.depthLimitReached: boolean or undefined
// result.cycleDetected: boolean or undefined

Depth Limit

// Default: MAX_DEPTH = 3
// Configurable via AGENT_MAX_DEPTH env var

Depth 0: User calls agent A
Depth 1: A calls agent B
Depth 2: B calls agent C
Depth 3: C calls agent D - REJECTED (depth exceeded)

Cycle Detection

The call stack is maintained per execution. If an agent at depth N calls an agent that is already in the stack at depth N, it's rejected as a cycle:

// A calls B (depth 1)
// B calls C (depth 2)
// C calls A (depth 3) - REJECTED: A already in stack at depth 0

Tool Registration

To enable call_agent within an agent's runtime:

const callAgentTool = NestedExecutor.registerCallAgentTool(nested);
// callAgentTool({ agent: "sub_agent_name", params: { ... } })

This would be wired into the runtime shim when executing nested agents, allowing agents to invoke each other as tools.

Timeout

Default execution timeout is 30 seconds per agent. Set via constructor or AGENT_TIMEOUT_MS env var.
